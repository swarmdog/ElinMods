# Elin Drug Empire Mod – Design Document

**Overview:** Elin (formerly *Elin’s Inn*) is a single-player settlement-building RPG with a built-in mod framework【3†L224-L232】.  In this mod, the player is given a new “Drug Empire” start option: they receive ample crafting and building items at launch (a “fast start”), but instead of the normal main quest, they will pursue an illicit drug-dealing career.  Drawing on *Drug Dealer Simulator*’s mixing-table crafting and Torn’s persistent crime-economy mechanics, the mod adds a new industry to Elin’s world.  All multiplayer interactions (buying, selling, reputation, etc.) are handled asynchronously via simple HTTP/HTTPS calls to a remote server, allowing multiple players to trade drug commodities in a shared economy while each runs their own Elin game. 

**Mod Support:**  Elin explicitly supports user-made mods【3†L224-L232】.  Mods are implemented as BepInEx plugin DLLs plus data files.  The development workflow is to write C# code (targeting **netstandard2.0**) that hooks into the game via Harmony patches, and to package it in an Elin `Package/ModName` folder.  For example, one community tutorial shows a minimal `.csproj` with references to `0Harmony.dll` and outputting to `Elin\Package\Mod_<ModName>\`【24†L149-L156】.  Every mod requires a `package.xml` metadata file in its folder【26†L468-L474】.  This XML defines the mod’s title, unique ID, author, game version and tags.  (For instance, `<title>My Elin Mod</title>` and `<id>my.unique.modid</id>`【27†L502-L511】.)  When placed under the game’s `Package` directory【26†L429-L438】 and launched in Elin’s Mod Viewer, the plugin is loaded and can be published to the Workshop【27†L594-L602】.

## Core Game Mechanics Analysis

- **Elin (Base Game):**  The base game is a **single-player colony-builder** set in a fantasy land called Ylva.  Players arrive in Tyris and “build and maintain a life” by developing a home base, recruiting NPCs, and crafting equipment【7†L170-L178】.  The standard main quest (e.g. “To the New Lands”) sends the player on story-driven tasks like investigating a cave and slaying a boss【7†L174-L183】.  Elin’s gameplay focuses on *base construction*, *crafting recipes* (e.g. building workbenches, tools, furniture, etc.), resource gathering (mining, farming), and hero quests (e.g. dungeon exploration for loot).  Mods can add new items, buildings, or alter logic using C# (via BepInEx/Harmony) or Excel-based “Source Sheets” (CWL tools) for data.  Elin’s asset pipeline uses XLSX sheets for items, feats, etc. and a Unity-based engine for runtime.

- **Drug Dealer Simulator (DDS):**  DDS is a 2020 single-player simulation where the player “creates and sells drugs, slowly creating a drug empire”【10†L132-L140】.  Key mechanics include: acquiring raw *substances* (e.g. baking soda, ibuprofen) by exploring or purchasing, then combining them on an in-game *mixing table* to synthesize drugs【10†L143-L152】.  The player then *sells* these drugs to customers.  High-quality drugs boost the player’s reputation in an area, opening access to more customers and territory【10†L149-L157】.  As the empire grows, the player must manage drug potency (poor mixtures can even kill clients) and avoid law enforcement.  In summary, DDS’s loop is **(collect ingredients → craft drugs → sell to customers → gain reputation)**, with risk of police.  

- **Torn (Torn City):**  Torn is a massively multiplayer, text-based crime RPG【12†L63-L68】.  Players engage in organized crimes, business, and trading in an online world with hundreds of thousands of players.  Key Torn elements: players can commit dozens of *crimes* for cash【31†L147-L150】, start and trade in companies and the stock market, buy/sell goods on a market or auction【31†L165-L168】, and use “drugs” as consumable items for buffs【31†L243-L247】.  The game runs continuously on servers (24/7) and players’ actions occur asynchronously.  Torn’s economy is player-driven: anyone can list items (including drugs) for sale, and thousands of active players can interact.  Notably, Torn emphasizes *asynchronous multiplayer*: every player has a persistent account that anyone can affect via official API or shared events.  

**Core Mechanic Mapping:**  Our mod will import the *DDS loop* into Elin: gather raw “chemical” resources, craft drugs at a new mixing station, and sell to clients to grow reputation.  It will mirror Torn’s *economy and trading* by enabling players to post buy/sell orders via a web server (handled asynchronously).  In Elin terms, we can treat drug ingredients as new Items (e.g. a custom material “Ephedrine” or “Herbal Extract”) and Drugs as equipment or consumables.  Crafting will use Elin’s recipe system: either define a new “Mixing Table” building or repurpose an existing crafting table with new recipes that yield drug items.  Selling could use Elin’s **CWL Merchant/Converter** framework to add new merchants or shops that trade drugs, or intercept player inventory to simulate “selling” and reward money.  Reputation might be tracked as an internal stat or achievement.  We also plan to *bypass* the normal quest chain: if the player selects the Drug mode, we will skip the Ashland/Nymelle quests and instead start the mod’s own drug industry objectives.

## Mod Structure and Components

- **New Starting Option:**  Add a custom UI option on the New Game screen (similar to the “Fast Start” workshop mod).  The player chooses “Drug Dealer” start.  In code, we intercept game initialization (via a Harmony patch on the main menu or game start logic) to provide alternate setup.  We will spawn the player’s character and base with predefined items: e.g. multiple crates of building materials, a mixing table built, and ingredient stash.  All tutorial quests and story triggers are disabled.  This is analogous to DDS’s early cash infusion, giving the player tools to immediately begin business.

- **Starting Items & Assets:**  Use Elin’s Source Sheets or CWL to add new item definitions for *drug ingredients* and *finished drugs*.  For example, create new entries in the “Materials” and “Potions” sheets for chemicals and substances.  Provide sprites (2D icons) for these items.  Also add a “Mixing Table” as a piece of furniture or workstation (similar to how a Carpenter’s Table or Alchemy Station works in vanilla).  The new building can be placed automatically at start or provided as a recipe.  In package resources, include any custom art (icons, sprites) for these new items.

- **Crafting Mechanics:**  Define crafting recipes (likely via SourceSheet Recipe or CWL) so that combining certain raw materials on the Mixing Table yields a “Drug” item.  The recipe can require e.g. 1 Plant Fiber + 1 Chemical = 1 “Herbal Stimulant”, or more complex combinations.  Alternatively, implement recipe logic in C# by patching the crafting system: detect when the player uses the Mixing Table and spawn a drug item with attributes (quality/reputation yield).  0Harmony patches may intercept the `CraftRecipe` call to add custom behavior (like adjusting output quality based on input).  The mod will also need to register these new recipes/blueprints with the game’s Crafting list.

- **Selling & Reputation:**  We plan to simulate selling by giving the player an interface to list drugs on a shared market.  When a player “sells” a drug (for example, by placing it on a special in-game interface or simply dumping it in their base), the mod will send an HTTP request to the server announcing the sale offer (quantity, quality, price).  The server will store these listings.  If another player later wants that drug, their game can fetch the list and “buy” via HTTP.  Upon a successful sale or purchase, both players’ mods update their local economy state (e.g. adjusting in-game currency or resources).  Reputation can be tracked per-area or per-player: selling higher-quality drugs yields more reputation points, unlocking access to richer areas or more clients.  This mirrors DDS’s reputation mechanic【10†L149-L157】.  Elin does not natively track “reputation,” so we will store it in the mod’s own data, perhaps using the game’s existing Achievement or Condition system (incrementing a hidden stat each sale).

- **Asynchronous Multiplayer:**  Because Elin is single-player by default, multiplayer is fully mod-based.  We will implement a simple REST API client in the C# plugin.  Using .NET’s `HttpClient` (or UnityWebRequest if needed), the mod will periodically poll the server or send events.  For example, every in-game day (or via a manual UI action), the game contacts the server: `GET /market/listings` to update available orders, `POST /market/offer` to post a new sale, `POST /trade/execute` to complete a transaction.  All network calls should use HTTPS for security.  The mod will run this HTTP client on a background thread or Unity coroutine to avoid stalling the game.  It will handle JSON responses (e.g. item type and price) and display them in a custom UI panel added to the game.  Players can browse and accept offers from others through this interface.  By communicating only through the central server, players play *asynchronously* – there is no direct real-time multiplayer, only shared market data. 

- **Elin Integration:**  We use BepInEx for hooking into the game.  The plugin’s `.cs` code will be built into a DLL and placed under `Package/ModName/` (with corresponding `.csproj` linking to Elin’s DLLs)【24†L149-L156】.  We will use 0Harmony to patch relevant methods: e.g. patch the game startup method to apply our “Drug Dealer” mode if selected, and patch any inventory/crafting functions for the mixing recipes.  We may also use CustomWhateverLoader (CWL) for bulk data import: CWL allows loading sheets of items or NPCs without code【24†L256-L260】, which could simplify adding new drugs as items.  For UI elements (like the market screen), we might use Unity’s UI via scripts or reuse existing in-game UI panels.  All new data files (images, XLSX definitions, JSON for data templates) go in the mod folder.

## Technical Specifications

- **Mod Folder Structure:**  Create a folder `<ElinDir>/Package/DrugEmpireMod/`.  Place `package.xml` here with fields for title (e.g. “Elin Drug Empire”), unique ID (`com.mymod.drugempire`), author, and target version (set it to the current Elin version)【26†L468-L474】【27†L531-L539】.  Include the built plugin DLL in `DrugEmpireMod/` or a subfolder (as per BepInEx convention).  Any custom textures (e.g. drug icons) go in this folder or a subfolder (e.g. `res/`).

- **Dependencies:**  The C# project will reference `Elin.dll` (the game assembly), `0Harmony.dll` for patching, and any networking libraries (standard .NET).  Set the project’s BaseOutputPath to the Elin Package path as in【24†L149-L156】.  We should also include `BepInEx.dll` references if needed.  For building, the project targets `netstandard2.0` (to match Elin’s .NET version)【24†L149-L156】.

- **Assets/Data:**  
  - *Item Data:* Define new rows in Elin’s “Materials” and “Potions” source sheets for raw ingredients and drugs.  For example, add “Chemicals” that are harvestable or purchasable from NPCs.  Add several drug items (“Speed Tonic”, “Herbal X”, etc.) with appropriate stats (e.g. worth a lot of currency, or provide a small buff on use).  
  - *Recipes:* In a Sheet or via CWL, add recipes for the Mixing Table.  Each recipe uses certain raw items and yields a drug (perhaps with variable quality if we implement that logic).  
  - *Building:* Add a new furniture item for “Mixing Table” (copying an existing table prefab or sprite).  Provide a recipe to build it (wood + materials) and optionally place one automatically in the player’s base at game start.  
  - *UI Prefabs:* If adding market/trading UI, prepare Unity prefabs or use a runtime-created GUI panel. Include any required sprites for buttons or icons.

- **Gameplay Scripts:**  
  - *Startup Logic:* A C# patch on the `GameStart` or `Intro` method to detect “Drug Mode” and give starting items (using the game’s `CreateItem` API).  Also disable any tutorial quest flags.  
  - *Mixing Logic:* Either add a new `Recipe` object (so the game’s crafting logic handles it), or patch the `CraftingStationController` to recognize our Mixing Table and perform custom output.  We may also need scripts to place the Mixing Table in the base.  
  - *Market Client:* Implement a `HttpClient` instance in the plugin, with methods like `PostNewListing(DrugListing)` and `GetMarketData()`.  These call our server’s REST API and parse JSON responses.  We will need C# data classes for serialization (e.g. `DrugListing { string itemId; int price; int quantity; }`).  
  - *In-Game Actions:* Scripts to add UI buttons (e.g. a “Market” button in the base UI) that open the trading panel.  Scripts to handle the “Sell Drug” action (remove item from inventory and send HTTP request) and “Buy Drug” action (give item to player, deduct currency).  Possibly use Elin’s `InventoryTransaction` or similar classes.  
  - *Data Storage:* Use Elin’s save system (e.g. `SaveData`) or local config to remember the player’s cash, inventory, and any mod-specific state (like reputation or active offers).

- **Server API (External):**  We will design simple endpoints on a web server (this is part of the design, not Elin code): e.g.:  
  1. `GET /api/market/listings` – returns a JSON list of all active drug listings (item, price, seller ID).  
  2. `POST /api/market/offer` – used when a player creates a sale offer (body includes player ID, item ID, quantity, price).  
  3. `POST /api/market/buy` – for when a player accepts an offer (specify listing ID and buyer ID). The server then transfers the offer to the buyer (and may notify the seller next time they poll).  
  4. `GET /api/player/status?playerId=` – (optional) returns a player’s current reputation or balance if we track it server-side.  

  The mod’s HTTP client will call these endpoints periodically (e.g. every in-game hour or on demand).  All communication uses JSON and HTTPS.  The server can be a simple Node.js/Python web app or cloud function storing data in a database.

- **Integration & Packaging:**  During development, build the C# project into the game’s `Package` folder as described【24†L149-L156】【26†L429-L438】.  Include the `package.xml` (with correct `<version>` matching the game’s version【27†L531-L539】).  When ready, the mod can be published via the Elin Workshop: open the Mod Viewer in-game, find “Drug Empire” mod as local, and press Publish【27†L594-L602】.  The Workshop page can use an image preview (e.g. a stylized drug icon) and tags like `Gameplay, NewMode`.

## Testing Checklist

1. **Build and Load:** Verify that the plugin DLL and `package.xml` appear in Elin’s Mod Viewer as “[Local]”. Launch the game and confirm that the mod loads without errors (check in logs).  
2. **New Game Menu:** Start a new game and select the “Drug Dealer” option. Ensure the main quest does not start and the player is in the base with the expected starting items (as per design).  
3. **Crafting:** Test that the Mixing Table can be built/placed, and that the custom recipes work. For each recipe, confirm the correct inputs produce the drug item.  
4. **Inventory & Economy:** Check that sold drugs are removed from inventory and the correct amount of currency is awarded. Test the “sell” process: the mod should send a listing to the server; verify this by inspecting the server database or API response.  
5. **Multiplayer Interaction:** Run two clients (or emulate) and perform a trade: one posts an offer on the market, the other retrieves it and buys. Verify that after the transaction, both clients’ states update correctly (items transferred, money deducted/added). Also test network failure or offline mode (mod should handle no-response gracefully).  
6. **Persistence:** Save and reload the game. Ensure mod state (cash, inventory, reputation, pending offers) is correctly preserved across sessions.  
7. **Compatibility:** Disable the mod and verify that Elin reverts to normal startup and main quests (no leftover data). Test with other mods installed to ensure no conflicts (especially other mods that patch the new-game screen or crafting).  
8. **Edge Cases:** Attempt invalid trades (buying nonexistent offer), crafting with wrong ingredients, and ensure the mod handles errors without crashing (showing error messages if needed). Confirm that the `package.xml` versioning works: if the game is updated, an older-versioned mod shows the warning and unloads【27†L538-L546】.

Each step should pass without critical bugs.  Logging and in-game debug messages (if implemented) will help verify internal states.  Success criteria: players can start the game in Drug mode, produce and trade drugs via the asynchronous market, and see their virtual drug empire grow in resources and reputation.
